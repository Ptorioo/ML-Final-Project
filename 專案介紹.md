# AI 生成影像偵測專案介紹

## 一、專案概述

本專案的主題是 **AI-Generated Image Detection**，也就是判斷一張輸入影像是真實影像，還是由 AI 模型生成的影像。隨著生成式 AI 技術快速發展，AI 生成圖片在視覺品質上越來越接近真實照片，因此如何建立一個能夠辨識真實影像與生成影像的分類模型，是一個具有實務價值的深度學習問題。

本專案目前的定位是建立一套可重現的 baseline 實驗流程。專案使用 CNN 與 ResNet 系列模型作為基準模型，針對真實圖片與 AI 生成圖片進行二元分類，並透過 accuracy、precision、recall、F1-score、AUROC 與 confusion matrix 等指標分析模型表現。

整體任務可定義為：

```text
Input:  一張 RGB 影像
Output: 該影像屬於 AI 生成影像的機率
Class 0: 真實影像
Class 1: AI 生成影像
```

專案中的標籤定義如下：

| 資料夾名稱 | 類別意義 | Label |
|---|---|---:|
| `REAL` | 真實影像 | 0 |
| `nature` | 真實影像 | 0 |
| `FAKE` | AI 生成影像 | 1 |
| `ai` | AI 生成影像 | 1 |

模型最後輸出一個 logit，訓練時使用 `BCEWithLogitsLoss`。在評估時，logit 會經過 sigmoid 轉換成 AI 生成影像的機率，若機率大於或等於 `0.5`，則預測為 AI 生成影像；否則預測為真實影像。

## 二、專案目標

本專案的主要目標包含以下幾點：

1. 建立真實影像與 AI 生成影像的二元分類 baseline。
2. 比較不同模型架構在 AI 生成影像偵測任務上的表現。
3. 支援不同資料集，包括 CIFAKE、TinyGenImage，以及兩者合併訓練。
4. 建立可重現的訓練流程，包含固定 seed、儲存實驗設定、訓練歷史與測試結果。
5. 使用多種評估指標觀察模型能力，不只看 accuracy，也分析 precision、recall、F1-score、AUROC 與 confusion matrix。
6. 作為後續更複雜偵測模型或多分支模型的實驗基礎。

## 四、使用資料集

### 4.1 CIFAKE

CIFAKE 是一個用於真實圖片與 AI 生成圖片分類的資料集。專案預期 CIFAKE 放置於 `dataset/cifake/`，結構如下：

```text
dataset/
  cifake/
    train/
      REAL/
      FAKE/
    test/
      REAL/
      FAKE/
```

其中：

- `REAL`：真實影像，label 為 `0`
- `FAKE`：AI 生成影像，label 為 `1`

在訓練時，程式會使用 CIFAKE 的 `train/` 作為原始訓練資料，並從中切出 validation set。預設 validation 比例為 `0.1`。切分方式是 stratified split，也就是會分別從真實類別與 AI 類別中抽取相同比例作為 validation，避免 validation set 類別比例失衡。

目前 CIFAKE 的資料切分如下：

| Split | 真實影像 | AI 生成影像 | 總數 |
|---|---:|---:|---:|
| Train | 45,000 | 45,000 | 90,000 |
| Validation | 5,000 | 5,000 | 10,000 |
| Test | 10,000 | 10,000 | 20,000 |

### 4.2 TinyGenImage

TinyGenImage 是另一個用於 AI 生成影像偵測的資料集。此資料集通常依照不同生成模型分成多個 generator 資料夾，例如 BigGAN、Stable Diffusion 等。專案預期的資料結構如下：

```text
dataset/
  tiny-genimage/
    imagenet_ai_0419_biggan/
      train/
        ai/
        nature/
      val/
        ai/
        nature/
    ...
```

其中：

- `nature`：真實影像，label 為 `0`
- `ai`：AI 生成影像，label 為 `1`

TinyGenImage 模式下，`train/` 會作為訓練資料，`val/` 會作為 validation 與 test 使用。使用者也可以透過 `--generators` 選擇特定 generator，例如只訓練某一類生成模型的影像。

目前 TinyGenImage 的資料切分如下：

| Split | 真實影像 | AI 生成影像 | 總數 |
|---|---:|---:|---:|
| Train | 14,000 | 14,000 | 28,000 |
| Validation / Test | 3,500 | 3,500 | 7,000 |

### 4.3 Combined Dataset

除了單獨使用 CIFAKE 或 TinyGenImage，本專案也支援 combined 模式，也就是把 CIFAKE 與 TinyGenImage 合併訓練。

combined 模式下的訓練資料組成如下：

| Split | 資料來源 | 真實影像 | AI 生成影像 |
|---|---|---:|---:|
| Train | CIFAKE | 45,000 | 45,000 |
| Train | TinyGenImage | 14,000 | 14,000 |
| Validation | CIFAKE | 5,000 | 5,000 |
| Validation | TinyGenImage | 3,500 | 3,500 |

因此 combined 模式的總訓練資料量為：

| Split | 真實影像 | AI 生成影像 | 總數 |
|---|---:|---:|---:|
| Train | 59,000 | 59,000 | 118,000 |
| Validation | 8,500 | 8,500 | 17,000 |

需要注意的是，目前訓練腳本在 combined 模式下，test set 仍然主要使用 CIFAKE test。因此 combined 實驗結果代表「使用 CIFAKE + TinyGenImage 訓練後，在 CIFAKE test 上的表現」，而不是完整的跨資料集共同測試。

## 五、資料前處理與資料增強

資料前處理由 `src/data_augmentation.py` 中的 `build_transform` 負責。所有圖片在進入模型前都會經過以下處理：

```text
1. 轉換為 RGB
2. Resize 成固定大小
3. 可選的資料增強
4. 轉換為 Tensor
5. Normalize
```

### 5.1 RGB 轉換

不同資料集中的圖片格式可能不完全一致，例如有些圖片可能是灰階或包含 alpha channel。因此專案會先將所有影像轉成 RGB，確保模型輸入固定為 3 個 channel。

### 5.2 Resize

預設輸入大小為 `32 x 32`，對應 CIFAKE 類似 CIFAR 的小圖設定。使用者可以透過 `--semantic-size` 調整輸入尺寸，例如：

```powershell
python src/train.py --dataset-root dataset --datasets tiny-genimage --semantic-size 224
```

若使用 ImageNet pretrained ResNet，通常會搭配 `224 x 224` 輸入尺寸與 ImageNet normalization。

### 5.3 Normalization

專案支援兩種 normalization：

| 模式 | Mean | Std |
|---|---|---|
| `cifar10` | `(0.4914, 0.4822, 0.4465)` | `(0.2470, 0.2435, 0.2616)` |
| `imagenet` | `(0.485, 0.456, 0.406)` | `(0.229, 0.224, 0.225)` |

預設使用 `cifar10` normalization。若使用 ImageNet pretrained ResNet，較適合改用：

```powershell
--normalization imagenet
```

### 5.4 資料增強

若加上 `--augment`，訓練資料會額外套用輕量資料增強：

```text
RandomHorizontalFlip(p=0.5)
RandomApply(GaussianBlur, p=0.25)
```

資料增強只會套用在 training set，不會套用在 validation 或 test set。這樣可以避免評估結果受到隨機增強影響。

## 六、模型架構

本專案目前支援三種模型：

```text
simple_cnn
resnet18
resnet50
```

### 6.1 Simple CNN

`simple_cnn` 是專案自定義的 baseline CNN。此模型使用多層 convolution block 擷取影像特徵，最後透過 fully connected layer 輸出二元分類 logit。

整體架構如下：

```text
Input Image
  ↓
Conv Block 1: 3 → 32 channels
  ↓
MaxPool
  ↓
Conv Block 2: 32 → 64 channels
  ↓
MaxPool
  ↓
Conv Block 3: 64 → 128 channels
  ↓
MaxPool
  ↓
Conv Block 4: 128 → 256 channels
  ↓
Adaptive Average Pooling
  ↓
Flatten
  ↓
Dropout
  ↓
Linear(256, 1)
  ↓
Output Logit
```

每個 convolution block 由兩層 convolution 組成：

```text
Conv2d
BatchNorm2d
ReLU
Conv2d
BatchNorm2d
ReLU
```

此模型的特點是：

- 架構簡單，適合作為 baseline。
- 訓練速度較快。
- 參數量相對 ResNet 較少。
- 對 CIFAKE 這類較規則的資料集已有不錯表現。
- 泛化能力可能不如更深的網路，尤其是在跨資料集或不同生成模型來源時。

### 6.2 ResNet-18

`resnet18` 使用 torchvision 提供的 ResNet-18 架構。ResNet 的核心概念是 residual learning，也就是透過 shortcut connection 讓模型學習 residual function。這可以減少深層網路訓練時的梯度消失問題，讓模型能夠更穩定地訓練。

標準 ResNet block 的概念如下：

```text
Input x
  ↓
Convolution Layers
  ↓
F(x)
  ↓
F(x) + x
  ↓
Output
```

這種 residual connection 讓模型不需要直接學習完整映射，而是學習輸入與輸出之間的差異，因此對深層模型訓練很有幫助。

在本專案中，ResNet-18 的最後分類層會被替換成：

```python
Linear(in_features, 1)
```

也就是只輸出一個 logit，用於二元分類。

### 6.3 ResNet-50

`resnet50` 同樣使用 torchvision 的 ResNet-50。與 ResNet-18 相比，ResNet-50 更深，且使用 bottleneck block，因此模型容量更大，理論上可以學到更複雜的影像特徵。

不過更大的模型也可能帶來以下問題：

- 訓練時間較長。
- 需要更多 GPU 記憶體。
- 若資料分布較單一，可能出現 overfitting。
- 在小尺寸輸入上，模型優勢不一定明顯。

目前專案支援 ResNet-50，但主要已完成結果集中在 Simple CNN 與 ResNet-18。

### 6.4 CIFAR-style ResNet Stem

原始 ImageNet ResNet 的開頭通常使用：

```text
7x7 convolution, stride 2
max pooling
```

這種設計適合 ImageNet 的 `224 x 224` 大尺寸圖片，但對 CIFAKE 這類 `32 x 32` 小圖而言，過早下採樣可能會造成空間資訊流失。

因此本專案預設會把 ResNet 的 stem 改成較適合小圖的形式：

```text
3x3 convolution, stride 1, padding 1
移除 max pooling
```

這讓模型在處理小尺寸影像時能保留更多細節。如果使用者想保留 ImageNet 原始 stem，可以加上：

```powershell
--imagenet-resnet-stem
```

## 七、訓練流程

專案的訓練流程如下：

```text
1. 解析 command line arguments
2. 固定 random seed
3. 建立 dataset 與 dataloader
4. 建立模型
5. 設定 loss function 與 optimizer
6. 逐 epoch 訓練
7. 每個 epoch 後進行 validation
8. 若 validation loss 下降，儲存 best checkpoint
9. 訓練完成後載入 best checkpoint
10. 在 test set 上進行最終評估
11. 儲存測試結果與錯誤樣本
```

### 7.1 Loss Function

本專案使用的 loss function 是：

```python
BCEWithLogitsLoss
```

此 loss 適合二元分類任務。它內部會結合 sigmoid 與 binary cross entropy，因此模型輸出不需要先經過 sigmoid。這樣在數值穩定性上通常比先 sigmoid 再算 binary cross entropy 更好。

### 7.2 Optimizer

本專案使用：

```python
AdamW
```

預設參數為：

```text
learning rate = 0.001
weight decay  = 0.0001
```

AdamW 是 Adam 的改良版本，將 weight decay 與 gradient update 解耦，常用於深度學習模型訓練。

### 7.3 Best Checkpoint 選擇

每個 epoch 結束後，模型會在 validation set 上計算 loss 與各項指標。若目前 epoch 的 validation loss 比之前最佳結果更低，就會儲存當前模型為：

```text
outputs/<experiment_name>/best_model.pt
```

因此，本專案的 best model 是根據 validation loss 選出，而不是根據 accuracy、F1-score 或 AUROC。

### 7.4 訓練輸出

每次訓練會在 `outputs/` 中建立對應實驗資料夾，例如：

```text
outputs/resnet18/
outputs/simple_cnn/
outputs/resnet18_combined/
```

每個實驗資料夾通常包含：

| 檔案 | 說明 |
|---|---|
| `config.json` | 本次實驗的模型、資料集、超參數設定 |
| `split_summary.json` | train / validation / test 的資料數量 |
| `history.json` | 每個 epoch 的 train 與 validation 指標 |
| `best_model.pt` | validation loss 最佳的模型權重 |
| `test_results.json` | test set 最終評估結果 |

## 十、評估指標

本專案使用多個指標評估模型表現。

### 10.1 Accuracy

Accuracy 表示整體分類正確率：

```text
Accuracy = 正確預測數 / 總樣本數
```

Accuracy 容易理解，但如果資料類別不平衡，單看 accuracy 可能會誤判模型表現。雖然本專案目前資料集大多是 real/fake 平衡，但仍搭配其他指標進一步分析。

### 10.2 Precision

Precision 表示模型預測為 AI 生成影像的樣本中，有多少真的為 AI 生成影像：

```text
Precision = TP / (TP + FP)
```

Precision 高表示模型較少把真實圖片誤判為 AI 生成圖片。

### 10.3 Recall

Recall 表示所有 AI 生成影像中，有多少被模型成功偵測出來：

```text
Recall = TP / (TP + FN)
```

Recall 高表示模型較少漏掉 AI 生成圖片。

### 10.4 F1-score

F1-score 是 precision 與 recall 的調和平均：

```text
F1 = 2 * Precision * Recall / (Precision + Recall)
```

當希望同時考慮誤判與漏判時，F1-score 是比 accuracy 更全面的指標。

### 10.5 AUROC

AUROC 衡量模型在不同 threshold 下區分正負類別的能力。AUROC 越接近 `1.0`，代表模型越能把真實圖片與 AI 生成圖片分開。

AUROC 的優點是它不只看固定 threshold `0.5` 的分類結果，而是看整體排序能力。因此在二元分類任務中，AUROC 是很重要的輔助指標。

### 10.6 Confusion Matrix

本專案使用的 confusion matrix 格式如下：

```text
                 Pred REAL    Pred FAKE
True REAL           TN          FP
True FAKE           FN          TP
```

其中：

| 縮寫 | 意義 |
|---|---|
| TN | 真實影像被正確預測為真實影像 |
| FP | 真實影像被錯誤預測為 AI 生成影像 |
| FN | AI 生成影像被錯誤預測為真實影像 |
| TP | AI 生成影像被正確預測為 AI 生成影像 |

## 十二、結果分析

### 12.1 CIFAKE 上的模型表現

在 CIFAKE 測試集上，`simple_cnn` 與 `resnet18` 都取得相當高的表現：

```text
simple_cnn accuracy: 0.9607
resnet18 accuracy:  0.9628
```

兩者 AUROC 都約為 `0.994`，代表模型能非常有效地區分 CIFAKE 中的真實圖片與 AI 生成圖片。

其中 `resnet18` 的表現略優於 `simple_cnn`：

```text
simple_cnn F1: 0.9611
resnet18 F1:  0.9631
```

不過差距並不大，表示在 CIFAKE 這個資料集上，即使是自定義的 simple CNN baseline，也已經能學到相當有效的判別特徵。

### 12.2 TinyGenImage 上的模型表現

TinyGenImage 的結果明顯比 CIFAKE 困難：

```text
simple_cnn_tiny-genimage accuracy: 0.6984
resnet18_tiny-genimage accuracy:  0.6916
```

這表示 TinyGenImage 的資料分布可能更複雜，或生成模型種類與影像內容更加多樣，使得模型較難單靠目前的 baseline 架構達到 CIFAKE 上的高準確率。

值得注意的是，`resnet18_tiny-genimage` 的 precision 較高，但 recall 較低：

```text
precision: 0.7493
recall:    0.5757
```

這代表 ResNet-18 在 TinyGenImage 上比較保守。當它預測某張圖片是 AI 生成時，正確率較高；但是它漏掉了較多 AI 生成圖片。

相對地，`simple_cnn_tiny-genimage` 的 recall 較高：

```text
precision: 0.6906
recall:    0.7189
```

這代表 simple CNN 抓到的 AI 生成圖片比較多，但也更容易把真實圖片誤判成 AI 生成。

### 12.3 Combined Training 的效果

`resnet18_combined` 使用 CIFAKE 與 TinyGenImage 合併訓練，最後在 CIFAKE test 上得到：

```text
accuracy: 0.9586
F1-score: 0.9591
AUROC:    0.9931
```

和單純使用 CIFAKE 訓練的 `resnet18` 相比：

```text
resnet18 accuracy:          0.9628
resnet18_combined accuracy: 0.9586
```

combined training 在 CIFAKE test 上略低一些。這可能是因為模型在訓練時同時看到 TinyGenImage 的資料分布，需要學習更廣泛的特徵，因此在單一 CIFAKE 測試集上的最佳化程度稍微下降。

不過，combined training 的目標通常不只是提升單一資料集表現，而是希望模型對不同來源的 AI 生成圖片有更好的泛化能力。因此若要完整評估 combined training 的價值，應該進一步測試：

- 在 CIFAKE 訓練、TinyGenImage 測試的 cross-dataset performance。
- 在 TinyGenImage 訓練、CIFAKE 測試的 cross-dataset performance。
- 在 combined 訓練後，分別於 CIFAKE 與 TinyGenImage 測試的結果。
- 對不同 generator 分開評估，觀察模型是否只對特定生成模型有效。

## 十三、Confusion Matrix 分析

以 `resnet18` 在 CIFAKE test 上的結果為例：

```text
TN = 9543
FP = 457
FN = 288
TP = 9712
```

這代表：

- 有 9,543 張真實圖片被正確判斷為真實圖片。
- 有 457 張真實圖片被誤判為 AI 生成圖片。
- 有 288 張 AI 生成圖片被誤判為真實圖片。
- 有 9,712 張 AI 生成圖片被正確偵測出來。

從這組結果可以看出，模型對 AI 生成圖片的偵測能力相當強，漏判數量比誤判數量更少。換句話說，模型比較傾向於抓出 AI 生成影像，但仍有少部分真實影像會被誤判為生成影像。

以實務應用來說，FP 與 FN 的重要性取決於使用情境：

- 若應用在內容審核，可能希望降低 FN，避免 AI 生成內容漏網。
- 若應用在攝影作品認證，可能希望降低 FP，避免真實作品被錯誤標記為 AI 生成。

因此未來可以依照應用需求調整 threshold，不一定固定使用 `0.5` 作為分類門檻。

## 十四、專案優點

本專案目前有以下優點：

1. **流程完整**  
   從資料讀取、前處理、模型建立、訓練、驗證、測試到結果儲存都有完整流程。

2. **可重現性佳**  
   訓練時會固定 random seed，並儲存 `config.json`，方便追蹤每次實驗設定。

3. **支援多資料集**  
   可使用 CIFAKE、TinyGenImage，也可以將兩者合併訓練。

4. **支援多模型比較**  
   目前支援 simple CNN、ResNet-18 與 ResNet-50。

5. **評估指標完整**  
   不只計算 accuracy，也包含 precision、recall、F1-score、AUROC 與 confusion matrix。

6. **保留錯誤樣本**  
   `test_results.json` 中會記錄部分分類錯誤樣本，方便後續進行 error analysis。

7. **實驗輸出清楚**  
   每次實驗都會有獨立資料夾，保存設定、訓練歷史、最佳 checkpoint 與測試結果。

## 十五、限制與可改進方向

雖然目前 baseline 已經能在 CIFAKE 上達到不錯表現，但仍有一些限制：

### 15.1 跨資料集泛化仍需更完整評估

目前 CIFAKE 上的結果很好，但 TinyGenImage 上的表現明顯下降。這代表模型可能學到部分資料集特定特徵，而不一定完全學到通用的 AI 生成痕跡。

未來可以加入更多 cross-dataset evaluation，例如：

```text
CIFAKE train → TinyGenImage test
TinyGenImage train → CIFAKE test
Combined train → CIFAKE test + TinyGenImage test
```

### 15.2 輸入尺寸與 normalization 可進一步調整

目前預設使用 `32 x 32` 輸入尺寸與 CIFAR-10 normalization。這對 CIFAKE 合理，但對 TinyGenImage 或更高解析度資料集而言，可能會損失影像細節。

未來可以比較：

```text
32 x 32
64 x 64
128 x 128
224 x 224
```

並搭配不同 normalization 設定觀察效果。

### 15.3 可加入更強的資料增強

目前 augmentation 較輕量，只包含 horizontal flip 與 Gaussian blur。未來可以嘗試：

```text
RandomCrop
ColorJitter
JPEG compression
RandomResizedCrop
AugMix
RandAugment
```

尤其 AI 生成影像偵測很容易受到壓縮、縮放、裁切與平台轉檔影響，因此加入更接近真實使用情境的 augmentation 可能有助於提升泛化能力。

### 15.4 可加入更進階模型

目前模型以 CNN 與 ResNet 為主。未來可以嘗試：

```text
EfficientNet
ConvNeXt
Vision Transformer
Swin Transformer
CLIP-based classifier
frequency-domain model
multi-branch detector
```

其中 frequency-domain method 對 AI 生成影像偵測特別值得嘗試，因為生成模型可能會在高頻訊號或頻譜分布上留下特徵。

### 15.5 可加入 threshold analysis

目前分類 threshold 固定為 `0.5`。但在實際應用中，可以依需求調整 threshold：

- 提高 threshold：減少真實圖被誤判成 AI 的情況。
- 降低 threshold：減少 AI 圖被漏判成真實的情況。

未來可以畫出 ROC curve、PR curve，並比較不同 threshold 下的 precision-recall tradeoff。

## 十六、結論

本專案完成了一套完整的 AI 生成影像偵測 baseline pipeline，能夠使用 CIFAKE 與 TinyGenImage 進行 real-vs-AI image classification。專案支援 Simple CNN、ResNet-18 與 ResNet-50，並提供完整的訓練、驗證、測試與結果彙整流程。

從目前實驗結果來看，模型在 CIFAKE 上表現相當良好，`simple_cnn` 與 `resnet18` 都能達到約 96% accuracy 與 0.994 AUROC，表示 baseline CNN 模型已能有效分辨 CIFAKE 中的真實影像與 AI 生成影像。然而，在 TinyGenImage 上，模型表現下降到約 69% accuracy，顯示 AI 生成影像偵測在跨資料集與跨生成模型情境下仍具有挑戰性。

整體而言，本專案成功建立了一個清楚、可重現、可擴充的基準系統。後續若要提升模型泛化能力，可以從跨資料集評估、更高解析度輸入、更強資料增強、frequency-domain 特徵，以及更進階的模型架構等方向進一步改進。
